## 確定要件

### システム概要

- **システム名**: DataAgent
- **概要**: Databricks Genieのような自然言語データ分析システム。チャット形式で自然言語の質問を入力すると、SQL/GraphQLクエリを自動生成・実行し、結果をグラフやテーブルで可視化する
- **対象ユーザー**: ITリテラシーの高低に関わらず利用可能（マニュアルなしで使える説明的なUI）
- **配布形態**: 社内利用のみ

### 機能要件（既存）

1. **自然言語チャット入力**: チャット形式のUIで自然言語の質問を入力
2. **SQL自動生成**: Claude API（Anthropic）を使用して自然言語からSQLを自動生成
3. **SQL表示**: 生成されたSQLをユーザーに表示（透明性確保）
4. **SQL実行**: SELECTクエリのみ実行可能（読み取り専用）
5. **結果可視化**: クエリ結果を以下の形式で表示
   - 棒グラフ
   - 折れ線グラフ
   - 円グラフ
   - テーブル（一覧表示、データ幅に合わせた自動列幅調整）
6. **グラフ種類自動判定**: LLMがデータ特性から最適なグラフ種類を自動提案
7. **クエリ履歴**: 過去の質問と結果を保存・再利用可能
8. **ストリーミング応答**: ChatGPTのように応答が逐次表示される
9. **スキーマ自動取得**: DB接続時にINFORMATION_SCHEMAからテーブル・カラム情報（テーブルコメント・カラムコメント含む）を自動取得しLLMコンテキストに渡す
10. **エラーハンドリング**: 不正SQL生成時はエラー表示+再試行ガイドを提示
11. **AI分析コメント（オンデマンド）**: クエリ結果に対して、ユーザーが「AIに分析させる」ボタンをクリックすることでLLMがデータの傾向・特徴・注目ポイントを分析し、コメントをストリーミング表示。100行以上の大量結果に対しては警告を表示し、トークン消費についてユーザーに確認を求める
12. **会話コンテキスト維持**: 同一会話内の過去のやり取り（生成SQL含む）をLLMに渡し、直前のSQLに対する修正依頼に対応

### 機能要件（今回の改修で追加）

13. **複数DB接続先管理**: 画面からDB接続先（接続名、ホスト、ポート、ユーザー名、パスワード、DB名、DB種別）を登録・編集・削除できる
14. **DB接続先選択**: ヘッダーのドロップダウンからDB接続先を切り替えられる。チャットは常に1つのDBに接続
15. **DB切替による会話分離**: DB接続先ごとに会話履歴が分離される。サイドバーの履歴は選択中のDBの会話のみ表示
16. **接続中DB表示**: ヘッダーのDB選択ドロップダウンに現在接続中のDB名を常時表示
17. **DB接続テスト**: DB接続先登録・編集時に「接続テスト」ボタンで接続可否を確認できる
18. **DB管理モーダル**: DB選択ドロップダウンの「管理」ボタンからモーダルダイアログでDB接続先のCRUD操作が可能
19. **初回起動ガイド**: DB接続先が未登録の場合、「まずDB接続先を登録してください」というウェルカム画面を表示
20. **デフォルトDB選択**: アプリ起動時、最後に使用したDB接続先を自動選択
21. **パスワード暗号化**: DB接続先のパスワードはAES-256-GCMで暗号化してSQLiteに保存
22. **スキーマキャッシュ**: DB選択時にスキーマ情報を取得しキャッシュ。チャットごとの取得は不要

### 機能要件（v1.1 GraphQL対応で追加）

28. **GraphQL接続先管理**: 接続先登録フォームのDB種別に「GraphQL」を追加。GraphQL選択時はエンドポイントURL入力に切替。認証なし
29. **GraphQLクエリ自動生成**: LLMがGraphQLスキーマに基づいてGraphQLクエリ（Queryのみ、Mutation禁止）を自動生成
30. **GraphQLスキーマ自動取得**: Introspection Query（`__schema`）でType/Field情報を自動取得しLLMコンテキストに渡す
31. **GraphQLスキーマキャッシュ**: DB接続と同様、GraphQL選択時にIntrospectionでスキーマを取得しキャッシュ
32. **GraphQL接続テスト**: Introspection Queryが正常に返ればテスト成功と判定
33. **GraphQL結果のフラット化**: LLMに対してフラットなデータ構造を返すクエリ生成を指示。ネストしたJSONもテーブル・グラフで可視化可能に
34. **GraphQL結果可視化**: GraphQLレスポンスもDB接続と同じく棒グラフ・折れ線・円グラフ・テーブルで表示
35. **GraphQL会話分離**: GraphQL接続先ごとに会話履歴が分離される（DB接続先と同様）
36. **GraphQLエラーハンドリング**: GraphQLのerrors配列でのエラーにも「質問を変えてみてください」等のガイド付きエラー表示

### 機能要件（v1.0 追加済み）

23. **バージョン表示**: ヘッダーにgitハッシュ+ビルド日付のバージョン文字列を表示
24. **LLMバックエンド表示**: ヘッダーに使用中のLLMバックエンド（Anthropic API / Amazon Bedrock）とモデル名を表示
25. **LLMバックエンド切替**: 環境変数 `USE_BEDROCK=true` でAmazon Bedrock経由に切替可能。未設定時はAnthropic API直接呼び出し
26. **テキスト応答（SQL不要）**: LLMがSQLを生成できない質問にはテキストのみで応答（エラーにしない）
27. **横スクロール制御**: SQLコード・グラフ・テーブルが横に長い場合、各コンポーネント内で個別にスクロール（画面全体の横スクロールは発生しない）

### 改修に伴う変更点

- .envでの固定DB接続設定は廃止（ANTHROPIC_API_KEY等のLLM関連設定は.envに残す）
- 既存の会話履歴は破棄（マイグレーション不要、DB再作成）
- Docker Compose構成は変更なし（web + MySQL + phpMyAdmin）
- 既存のSQLバリデーション（SELECTのみ許可）はそのまま維持
- DB接続先数に上限なし
- DB接続先削除時は関連する会話履歴も一緒に削除
- テーブル制限機能は不要（接続先DBの全テーブルが対象）
- MySQLスキーマ取得時にutf8mb4でコメント情報を正しく取得（文字化け防止）
- スキーマ情報のカラム型にCOLUMN_TYPE（varchar(255)等）を使用（DATA_TYPEより詳細）
- チャット入力: Shift+Enterで送信、Enterで改行

### 非機能要件

- **同時利用ユーザー数**: 1-10人（小規模）
- **対応ブラウザ**: Chrome のみ
- **対応デバイス**: PCのみ（レスポンシブ不要）
- **UIテーマ**: ライトモードのみ
- **言語**: 日本語のみ
- **認証**: なし
- **エクスポート**: 不要（画面表示のみ）

### 技術選定

| カテゴリ | 選定技術 | 選定理由 | 却下した代替案 |
|---------|---------|---------|---------------|
| フロントエンド | React + TypeScript | グラフ描画ライブラリが豊富、エコシステムが成熟 | Vue.js, Next.js |
| バックエンド | Node.js + Express + TypeScript | フロントエンドと言語統一、型安全 | Python + FastAPI, Go |
| LLM | Claude API (Anthropic) / Amazon Bedrock | SQL生成精度が高く、APIがシンプル。Bedrock切替でAWS内完結可 | Gemini, GitHub Copilot |
| データベース | PostgreSQL / MySQL（両対応） | ユーザーの既存DB環境に合わせて選択可能 | 単一DB限定 |
| GraphQLクライアント | node-fetch（標準fetch） | Introspection + クエリ実行。追加ライブラリ不要 | graphql-request, Apollo Client |
| グラフライブラリ | Recharts | Reactネイティブで宣言的API、カスタマイズ容易、MITライセンス | Chart.js |
| デプロイ | Docker Compose | コンテナで一括起動、小規模に最適 | クラウドサービス |
| 暗号化 | Node.js crypto (AES-256-GCM) | 標準ライブラリで追加依存なし、十分なセキュリティ | bcrypt, argon2 |

### 接続先設定

- 画面から登録（.envでの固定設定は廃止）
- 接続情報はSQLiteに保存（DB接続のパスワードはAES-256-GCM暗号化）
- 暗号化キーは環境変数 `DB_ENCRYPTION_KEY` で管理
- MySQL / PostgreSQL / GraphQL に対応
- DB接続: リードオンリー制約はSQLバリデーション（SELECTのみ許可）で担保
- GraphQL接続: Queryのみ許可（Mutationは禁止）。認証なし
- GraphQL接続先はエンドポイントURL のみで登録（ホスト/ポート/DB名/ユーザー/パスワードは不要）

### UIレイアウト

- ヘッダー: DataAgentロゴ + バージョン表示 + LLMバックエンド表示 + 接続先選択ドロップダウン + 管理ボタン + 新しい会話ボタン
- ドロップダウン表示: `接続名 (mysql)` / `接続名 (postgresql)` / `接続名 (graphql)`
- サイドバー+メイン構成
  - 左サイドバー: 選択中の接続先の会話履歴一覧
  - 右メイン: チャット画面 + グラフ/テーブル表示エリア
- 接続先管理モーダル: ドロップダウンの「管理」ボタンからアクセス
  - DB種別で「GraphQL」選択時: ホスト/ポート/ユーザー/パスワード/DB名のフィールドを非表示にし、エンドポイントURL入力フィールドを表示

### スコープ外（将来対応）

- OpenAPI連携
- GraphQL認証（Bearer Token、API Key等）
- ダークモード
- レスポンシブデザイン（スマホ対応）
- 結果のエクスポート（CSV/画像）
- DB接続先ごとのテーブル制限機能

## 専門家分析

### PO分析

- MVPスコープを明確に定義: 自然言語→SQL/GraphQL→グラフの基本フローに集中
- v1.0改修: 複数DB接続先管理機能の追加
- v1.1改修: GraphQL接続先の追加。既存DB接続フローと同一UXで統合
- DB/GraphQL切替による会話分離はユーザー体験の核心。接続中表示で混乱を防止
- GraphQLのMutation禁止はDBのSELECT制限と同じ安全方針の延長

### Architect分析

- v1.1影響範囲: フロントエンド（登録フォームの動的切替）、バックエンド（GraphQL実行サービス追加、スキーマ取得のIntrospection対応）、LLMプロンプト（GraphQLクエリ生成用システムプロンプト追加）
- db_connectionsテーブル: db_typeに'graphql'追加、endpoint_urlカラム追加。host/port/username/password/database_nameはGraphQL時はNULL許容
- GraphQLクライアント: Node.js標準のfetchでIntrospection + クエリ実行。追加ライブラリ不要
- スキーマキャッシュ: DB接続と同じメモリキャッシュ機構を再利用。Introspection結果をLLMコンテキスト向けに整形
- LLMプロンプト切替: 接続先のdb_typeに応じてSQL生成用/GraphQL生成用のシステムプロンプトを切替
- GraphQLレスポンスのフラット化: LLMへのプロンプトでフラットなデータ構造を返すクエリ生成を指示
- OSSライセンス: 追加ライブラリなし（fetchは標準API）

### QA分析

- 回帰テスト重点: 既存のDB接続→SQL生成→グラフ描画フローが影響を受けないこと
- GraphQLテスト: Introspection→クエリ生成→実行→結果表示の一連フロー
- 接続テスト: GraphQLはIntrospection成功/失敗、DB接続は既存テスト維持
- フォーム切替テスト: DB種別変更時にフィールドが正しく切り替わること
- 会話分離テスト: GraphQL接続先とDB接続先で会話が混在しないこと
- エラーハンドリング: GraphQLのerrors配列パターンの検証

### Security分析

- DB接続: パスワードはAES-256-GCM暗号化保存。暗号化キーは環境変数管理
- DB接続: SQLバリデーション（SELECTのみ許可）維持
- GraphQL接続: Queryのみ許可（Mutationキーワードを検出して拒否）
- GraphQL接続: 認証なし（v1.1スコープ）。将来的にBearer Token等の認証対応予定
- GraphQLエンドポイントURLのバリデーション（HTTPSスキーマ検証等）
- Introspection結果の検証（不正レスポンスのハンドリング）
