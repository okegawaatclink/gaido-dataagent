## 確定要件

### システム概要

- **システム名**: DataAgent
- **概要**: Databricks Genieのような自然言語データ分析システム。チャット形式で自然言語の質問を入力すると、SQLを自動生成・実行し、結果をグラフやテーブルで可視化する
- **対象ユーザー**: ITリテラシーの高低に関わらず利用可能（マニュアルなしで使える説明的なUI）
- **配布形態**: 社内利用のみ

### 機能要件

1. **自然言語チャット入力**: チャット形式のUIで自然言語の質問を入力
2. **SQL自動生成**: Claude API（Anthropic）を使用して自然言語からSQLを自動生成
3. **SQL表示**: 生成されたSQLをユーザーに表示（透明性確保）
4. **SQL実行**: SELECTクエリのみ実行可能（読み取り専用）
5. **結果可視化**: クエリ結果を以下の形式で表示
   - 棒グラフ
   - 折れ線グラフ
   - 円グラフ
   - テーブル（一覧表示、データ幅に合わせた自動列幅調整）
6. **グラフ種類自動判定**: LLMがデータ特性から最適なグラフ種類を自動提案
7. **クエリ履歴**: 過去の質問と結果を保存・再利用可能
8. **ストリーミング応答**: ChatGPTのように応答が逐次表示される
9. **スキーマ自動取得**: DB接続時にINFORMATION_SCHEMAからテーブル・カラム情報（テーブルコメント・カラムコメント含む）を自動取得しLLMコンテキストに渡す
10. **エラーハンドリング**: 不正SQL生成時はエラー表示+再試行ガイドを提示
11. **AI分析コメント**: クエリ結果に対してLLMがデータの傾向・特徴・注目ポイントを自動分析し、コメントをストリーミング表示
12. **会話コンテキスト維持**: 同一会話内の過去のやり取り（生成SQL含む）をLLMに渡し、直前のSQLに対する修正依頼に対応

### 非機能要件

- **同時利用ユーザー数**: 1-10人（小規模）
- **対応ブラウザ**: Chrome のみ
- **対応デバイス**: PCのみ（レスポンシブ不要）
- **UIテーマ**: ライトモードのみ
- **言語**: 日本語のみ
- **認証**: なし
- **エクスポート**: 不要（画面表示のみ）

### 技術選定

| カテゴリ | 選定技術 | 選定理由 | 却下した代替案 |
|---------|---------|---------|---------------|
| フロントエンド | React + TypeScript | グラフ描画ライブラリが豊富、エコシステムが成熟 | Vue.js, Next.js |
| バックエンド | Node.js + Express + TypeScript | フロントエンドと言語統一、型安全 | Python + FastAPI, Go |
| LLM | Claude API (Anthropic) | SQL生成精度が高く、APIがシンプル | Gemini, GitHub Copilot |
| データベース | PostgreSQL / MySQL（両対応） | ユーザーの既存DB環境に合わせて選択可能 | 単一DB限定 |
| グラフライブラリ | Recharts | Reactネイティブで宣言的API、カスタマイズ容易、MITライセンス | Chart.js |
| デプロイ | Docker Compose | コンテナで一括起動、小規模に最適 | クラウドサービス |

### DB接続設定

- 環境変数で設定（.envファイル）
- 単一DB接続（複数DB切替不要）
- リードオンリー権限のDBユーザーで接続（Security要件）

### UIレイアウト

- サイドバー+メイン構成
  - 左サイドバー: クエリ履歴一覧
  - 右メイン: チャット画面 + グラフ/テーブル表示エリア

### スコープ外（将来対応）

- GraphQL/OpenAPI連携
- ダークモード
- レスポンシブデザイン（スマホ対応）
- 結果のエクスポート（CSV/画像）
- 複数DB接続先の管理
- LLMプロバイダーの切り替え機能

## 専門家分析

### PO分析

- MVPスコープを明確に定義: 自然言語→SQL→グラフの基本フローに集中
- GraphQL/OpenAPI連携は将来対応としてバックログに記録
- エクスポート機能は不要と確認。画面表示のみでMVPとして十分
- クエリ履歴は必要。ユーザー体験の核心部分

### Architect分析

- React + TypeScript / Node.js + Express の統一スタック
- Claude APIでSQL生成。ストリーミングレスポンス対応が必要
- Rechartsでグラフ描画。棒・折れ線・円・テーブルの4種類
- DB接続はPostgreSQL/MySQL両対応。抽象化レイヤーが必要（例: knex.js等のクエリビルダー）
- スキーマ情報はINFORMATION_SCHEMAから自動取得
- OSSライセンス: 社内利用のみのため、GPL/AGPLの伝播リスクは低い。ただし使用ライブラリはMITが望ましい

### QA分析

- テスト対象ブラウザ: Chrome のみ（テスト工数削減）
- SQL生成の正確性テスト: 代表的なクエリパターンでE2Eテスト
- エラーハンドリング: 不正SQL時の表示・ガイド確認
- ストリーミングの安定性テスト

### Security分析

- 認証なしだが、DB接続はリードオンリー権限に制限
- SQL Injection対策: LLM生成SQLをそのまま実行するリスクあり。SELECT文のみ許可するバリデーション必須
- Claude API キーは環境変数で管理（ソースコードに含めない）
- DB接続情報も環境変数で管理
- 社内ネットワーク内での利用を前提とするが、CORS設定は適切に行う
